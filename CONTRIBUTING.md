## How to Contribute

### Reporting Issues

- Check if the issue already exists
- Use the issue templates provided
- Include as much detail as possible

### Suggesting Features

- Open a feature request issue
- Describe the use case clearly
- Explain why this would be valuable

### Submitting Code

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m 'feat: add amazing feature'`
6. Push to your fork: `git push origin feature/amazing-feature`
7. Open a Pull Request

### Commit Message Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

### Code Style

- Follow existing code style
- Keep functions focused and small
- Add comments for complex logic
- Update documentation as needed

### Testing

- Test your changes with real projects
- Include examples in your PR description
- Document any breaking changes

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/architect-coordinator.git
cd architect-coordinator

# Run setup
bash scripts/setup.sh

# Run demo
bash scripts/demo.sh
```

## Documentation

- Update README.md for user-facing changes
- Update SKILL.md for AI behavior changes
- Add examples to INTEGRATION.md if relevant
- Keep QUICKREF.md concise

## Questions?

Open an issue with the `question` label or reach out to the maintainers.
